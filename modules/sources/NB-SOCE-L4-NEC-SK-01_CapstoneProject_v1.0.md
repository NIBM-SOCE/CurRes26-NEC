## 2. The Common Frame (master-issued, reproduced verbatim)

> **Capstone Project (5 credits, Level 4, Profile C - 250 notional hours):** Project briefings and methods input 15; Supervision meetings 21; Laboratory / build access (supervised) 39; Presentation, demonstration and viva 9; Independent project work 166. Assessment components: CW project artefact with build log (take-home, supervised access; oral vehicle: Presentation at demonstration) and a final report with **Viva-defence** (mandatory oral vehicle). No examination. Weightings proposed by the thread and confirmed at sign-off. The artefact is the second designed public-portfolio vehicle (with Creative Project) and is TVEC-orientable evidence.

## 3. Module Purpose and Aims

The capstone is where the pathway's separate competencies become one delivered thing. Over a supervised applied project the student plans, builds and defends a piece of work that integrates networking, security and systems competencies drawn from across the two years, develops the academic-writing and research-readiness the Level 6 research chain assumes (the M7 carry, weighted here), and produces a portfolio artefact that speaks to employers and to TVEC alike. The module teaches no new subject content: its substance is the integration, the discipline of delivery, and the defence.

## 5. Skills Programme (Profile C component structure)

Per the Skills variant, the lecture syllabus is replaced by the Profile C component structure; all timetabled figures per 05_Hours_Model.md v1.2 and the common frame above. Totals close at 250.

| Component | Structure | Hours |
|---|---|---|
| Project briefings and methods input | Five 3-hour blocks: project scoping, research and academic-writing methods, build and laboratory practice, evaluation and defence preparation, and portfolio presentation | 15 |
| Supervision meetings | Scheduled individual supervision across the project lifecycle | 21 |
| Laboratory / build access (supervised) | Supervised access to laboratory and build environments, including the virtualisation substrate where a project requires isolation | 39 |
| Presentation, demonstration and viva | The demonstration with Presentation oral vehicle and the final Viva-defence | 9 |
| Independent project work | Student-led project delivery: build, test, write-up and portfolio preparation | 166 |
| | Total | 250 |

**Pathway flavour (briefs, domains, supervision - not part of the master frame):** project briefs are drawn from the pathway's domains - enterprise networking, security operations, offensive-and-defensive validation, identity and data protection, virtualisation and cloud - so that each capstone integrates a defensible slice of the two-year spine. Supervision is matched to the project domain (D7). Offensive-security projects inherit the Ethical Hacking and Application Security ethical and legal framing in full, and are confined to controlled, authorised environments.
